交易复盘系统 - 数据库表结构文档

1. 系统概述

本系统主要围绕加密货币交易复盘构建，核心数据流向为：通过用户授权的 ApiKey，利用定时任务或主动触发获取交易所的原始 Trade（成交记录），再通过内部引擎将其聚合为具有完整开平仓生命周期和盈亏数据的 Leg（头寸/持仓），最终供用户打上 Tag（标签）进行高阶复盘与总结。

2. 实体关系图 (纯文本 ER 树)

[User] (核心用户主表)
  │
  ├─ 1对1 ── [UserSettings] (用户偏好配置)
  │
  ├─ 1对多 ── [Tag] (自定义复盘标签)
  │            │
  │            └─ 多对多 ── 关联 ── [Leg]
  │
  ├─ 1对多 ── [ApiKey] (绑定的交易所API凭证)
  │            │
  │            ├─ 1对多 ── [SyncLog] (每次拉取数据的审计日志)
  │            │
  │            └─ 1对多 ── [Trade] (从交易所拉取回来的原始订单)
  │                          │
  └─ 1对多 ── [Leg] (聚合头寸/复盘核心单元) 
               │             │
               └─ 包含多笔 ──┘

3. 核心表结构详解

3.1. 用户与认证模块

User (用户主表)

存储系统所有用户的核心认证与基础信息。

字段名 数据类型 约束/默认值 描述说明
id Int PK, AutoInc 内部主键
uuid String Unique, cuid 外部暴露的唯一标识
email String Unique 登录邮箱
passwordHash String Nullable 密码哈希
timezone String Default "UTC" 用户首选时区，用于K线时间对齐
status Enum Default ACTIVE 账户状态 (ACTIVE/SUSPENDED/DELETED)
role Enum Default USER 权限角色 (USER/ADMIN/PREMIUM)

UserSettings (用户偏好配置表)

存储用户的 UI 设置和全局偏好。

字段名 数据类型 约束/默认值 描述说明
userId Int Unique, FK 关联的用户 ID
defaultTimeframe String Default "30d" 仪表盘默认展示时间范围
currency String Default "USDT" 全局计价/显示货币
theme String Default "dark" UI 主题模式

3.2. 交易所数据接入模块

ApiKey (API 凭证表)

管理用户绑定的各大交易所 API 密钥。

字段名 数据类型 约束/默认值 描述说明
userId Int FK 归属的用户 ID
name String - 备注名（如：币安主账户）
exchange Enum - 交易所标识 (BINANCE, OKX 等)
apiKey String - API 公钥 (加密存储)
apiSecret String - API 密钥 (加密存储)
isVerified Boolean Default false 是否已通过连通性验证
syncStatus Enum Default PENDING 当前同步状态 (PENDING/SYNCING/COMPLETED/FAILED)

SyncLog (数据同步日志表)

记录每次后台通过 API 抓取数据的成功与失败明细，用于审计与排错。

字段名 数据类型 约束/默认值 描述说明
apiKeyId Int FK 关联的 API Key ID
status Enum - 任务最终运行状态
tradesImported Int Default 0 本次成功写入的新原始交易数
legsCreated Int Default 0 本次运算新创建的头寸数量
errorMessage String Nullable, Text 抓取失败时的错误信息简述

3.3. 核心交易复盘引擎模块

Trade (原始成交记录表)

直接从交易所接口拉取回来的不可变底稿数据。这是系统的数据基石。

字段名 数据类型 约束/默认值 描述说明
id String PK 交易所返回的真实 Trade ID
apiKeyId Int FK 抓取来源的 API Key
legId Int FK, Nullable 挂载到的聚合头寸(Leg) ID
symbol String - 交易对 (如 "BTC/USDT:USDT")
side Enum - 交易方向 (BUY / SELL)
price Float - 实际成交价格
amount Float - 成交的资产数量 (Base Asset)
fee Float - 交易手续费数量
timestamp DateTime - 交易所记录的订单发生时间

Leg (聚合持仓/头寸表)

由底层 Trade 经算法归集而来的完整交易生命周期，是用户复盘的核心单元。

字段名 数据类型 约束/默认值 描述说明
userId Int FK 冗余用户ID，便于顶层权限隔离与查询
symbol String - 交易对名称
side Enum - 仓位方向 (LONG / SHORT)
status Enum Default OPEN 状态 (OPEN/CLOSED/LIQUIDATED 等)
openDate DateTime - 首次建仓时间
openPrice Float - 首笔开仓成交价
averageEntry Float - 动态加权平均开仓成本价
averageExit Float Nullable 动态加权平均平仓出场价
realisedPnL Float Default 0 已实现盈亏 (原始计价币种)
netPnL Float Default 0 最终净利润 (已扣除手续费)
peakSizeUsd Float Nullable 持仓期间达到的最大名义价值
mae / mfe Float Nullable 最大不利波动 / 最大有利波动
strategy String Nullable 记录：使用的交易策略名称
mistakes String Nullable, Text 记录：犯错反思与总结

3.4. 辅助分类模块

Tag (标签表)

用于给交易 (Leg) 打上策略、情绪、形态等维度的自定义标签。

字段名 数据类型 约束/默认值 描述说明
userId Int FK, Nullable 所属用户（为空代表系统全局预设）
name String - 标签显示名称
slug String Unique URL 友好的唯一标识符
color String Default "#6366f1" 标签 UI 颜色代码 (Hex)
usageCount Int Default 0 冗余：被关联次数，用于前端热度排序

3.5. 系统级模块

SystemConfig (系统全局配置表)

存储站点的动态开关和环境变量配置。

字段名 数据类型 约束/默认值 描述说明
key String Unique 配置键名 (如 "MAINTENANCE_MODE")
value Json - 配置项的具体动态键值
description String Nullable, Text 作用及修改说明

4. 关键架构设计说明

  1. 不可变底稿原则：Trade 表的数据一旦从交易所拉取并去重入库，即视为财务流水底稿，系统业务层不应对其进行修改。所有复杂的盈亏计算均在 Leg 表中体现。
  2. 读写分离与冗余：Leg 表中冗余了 userId 字段，这是为了避免在查询用户历史复盘数据时，需要执行 User -> ApiKey -> Trade -> Leg 这样低效的跨表多级联查。
  3. 高阶指标预留：Leg 表中提前预留了 mae、mfe、entryQuality 等字段，需在后续引入 K 线数据源后，通过异步任务计算填入，计算结果将直接反映交易的真实抗风险与盈利能力。
